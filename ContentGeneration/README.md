# ContentPilot

ContentPilot is a local-first, agentic content generation platform for planning, generating, formatting, and reviewing social content across multiple channels.

## Overview

- Multi-agent pipeline using LangGraph (Research, Summarize, Writer, Formatter, Quality).
- Advanced RAG: Parent-child chunking, trafilatura web scraping, cross-encoder re-ranking, and query expansion.
- Weekly planning with calendar-level date overrides.
- Two generation modes:
  - Pre-generated research mode (web research through SearXNG).
  - Source dump mode (async parallel map-reduce summarization).
- Platform-aware output formatting (LinkedIn, Twitter/X, WhatsApp, Telegram).
- Real-time generation progress via SSE streams.
- Review queue with edit and refinement workflow.

### Persistent Knowledge Retrieval (pgvector)
ContentPilot uses **PostgreSQL with pgvector** to persistently store chunked embeddings for each channel. 
- **Why we use it**: Previously, documents were re-scraped and re-embedded into an ephemeral/in-memory ChromaDB on every single generation run. This was extremely slow and wasted API/compute resources.
- **How it helps**: 
  - **Deduplication**: We now hash chunk content (`content_hash`) to ensure the exact same scraped text is never embedded twice.
  - **Channel Memory**: Every article or search result generated by a channel is saved to the `embeddings` table permanently. Over time, the channel builds an enormous "second brain" of domain knowledge it can instantly draw upon for future posts, without needing to re-search the web.
  - **Speed**: Subsequent generations on similar topics retrieve context instantly from the database.

## Current Tech Stack

- Backend: FastAPI, LangGraph, SQLAlchemy, Pydantic, PostgreSQL + **pgvector**, httpx, trafilatura.
- Frontend: React + Vite.
- AI runtime: Ollama (local model serving).
- Search: SearXNG (Docker).
- Vector DB: PostgreSQL with pgvector (Docker).

## Repository Layout

- `backend/`: FastAPI app, LangGraph agents, services, data models.
- `frontend/`: React app and UI components.
- `docker-compose.yml`: Local multi-service setup (SearXNG + Postgres w/ pgvector).
- `init-scripts/`: startup scripts (including pgvector init helper).

## Quick Start

### 1. Prerequisites

- Python 3.10+
- Node.js 18+
- Ollama running locally
- Docker Desktop (Required for Postgres+pgvector and SearXNG)

### 2. Start Infrastructure (Database & Search)

```bash
docker compose up -d postgres
# Also start searxng if you want web research
# docker compose up -d searxng
```

### 3. Start Backend

```bash
cd backend
uv sync
uvicorn src.backend.main:app --reload --port 8000
```

### 3. Start Frontend

```bash
cd frontend
npm install
npm run dev
```

### 4. Optional: Start SearXNG Manually

```bash
docker-compose up -d searxng
```

Open the frontend on the URL printed by Vite (typically `http://localhost:5173`).

## How the Flow Works

1. Configure runtime settings from the app.
2. Create or select a channel profile.
3. Plan content in weekly templates and date-level overrides.
4. Generate content for a day or week.
5. Review, refine, and export platform-ready output.
